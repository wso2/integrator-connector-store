##  Additional Benefits of On-Demand Enrichment

1. **Faster filtering**: When user searches "Salesforce", only enrich matching results
2. **Faster sorting**: When user sorts by name, don't need pull counts immediately
3. **Reduced API load**: Ballerina Central API will thank you
4. **Better error handling**: Failures only affect visible items
5. **Progressive enhancement**: Page works even if enrichment fails

##  Trade-offs to Consider

### On-Demand Enrichment:
- **Pro**: Much faster initial load
- **Con**: Need to enrich when user navigates pages (mitigated by prefetching)
- **Con**: Slightly more complex code

### Caching:
- **Pro**: Near-instant repeat loads
- **Con**: Stale data (mitigated by 6-hour expiry + background refresh)
- **Con**: localStorage size (minimal - ~50KB for 800 items)
